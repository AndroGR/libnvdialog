A cross platform solution for dialogs.

Ever wanted a library that would handle all dialog boxes for you, regardless of the project, painlessly and efficiently?

NvDialog is an open-source, easy-to-use and rapidly developing library to display
dialog boxes on Windows, MacOS, Linux and other Unix-like OSes written in C, with C++
support as well. It was originally developed exclusively for games. Since then, NvDialog
has expanded its abilities to all sorts of uses.

Native UI frameworks

NvDialog uses the host's native UI framework or the standard library for the
job. For Windows, the WinAPI is used. On Linux, libadwaita
is used. On MacOS and other Unix-like OSes, Gtk3 is used instead. This means applications always follow the system preferences and the light / dark preferences.

Cross-platform - Never worry about bug testing again.

NvDialog is designed to work with multi-platform support in mind.
An idea behind NvDialog is Write once, compile everywhere.

That means, using NvDialog, your code will only need to be written once, without having
to detect the platform, while still being able to use that code on virtually every platform.
