A cross platform solution for dialogs.

Want to display dialog boxes in your new game, your command line utility,
or the setup program you're writing? This library is for you.

nvdialog is a simple, blazingly fast library for displaying window dialogs using the host's toolkit.
It works natively with C and C++ and also provides Rust bindings if desired.

Native UI frameworks

The library uses the host's native UI framework or the standard library for the
job. For Windows, the Win32 API is used. On GNU/Linux, Gtk3 is used with libadwaita
support coming soon. MacOS support is also being developed.

Cross-platform, but actually.

The library is designed to work with multi-platform support in mind, and
great effort has been made to ensure this can be achieved.
nvdialog currently supports the following platforms, tested and verified:
- GNU/Linux
- Windows 10
- Termux

Support is coming for these platforms soon:
- FreeBSD
- MacOS X

Multi-language support

The library supports C and C++ out of the box. Rust bindings are also on the
way that will allow Rust projects to safely use the library. Python support
will also be added in later versions if requested.

If your language isn't here, open a new issue on the GitHub Repository
requesting the language support to be added.
